# Advanced Zustand Skill

이 스킬은 Zustand를 활용하여 효율적인 전역 상태 관리를 구축하고, 특히 Next.js 환경에서 발생할 수 있는 하이드레이션 오류와 불필요한 리렌더링을 제어하기 위한 가이드라인을 정의합니다.

## 1. 불변성 관리 (Immer Middleware)
- **복잡한 객체**: 깊이가 있는 객체나 배열을 업데이트할 때는 `immer` 미들웨어를 사용하여 직접적인 할당 방식을 유지하면서 불변성을 보장합니다.
- **예시**: `set((state) => { state.user.profile.name = 'New Name'; })` 형식으로 가독성을 높입니다.

## 2. Next.js 하이드레이션 불일치 방지
- **이유**: 서버에서 생성된 HTML과 클라이언트의 Zustand 초기 상태가 다를 때 `Hydration Mismatch` 오류가 발생할 수 있습니다.
- **해결 (Hydration Guard)**: `useEffect`를 활용하여 컴포넌트 마운트 전에는 `null`을 반환하거나 기본 스켈레톤을 보여주는 방식, 혹은 `useStore` 커스텀 훅을 사용하여 마운트 후 상태를 읽어오도록 설계합니다.

## 3. 리렌더링 최적화 (Selective Subscription)
- **셀렉터 사용**: `useStore(state => state.foo)`와 같이 필요한 상태만 선택하여 구독함으로써 관련 없는 상태 변경으로 인한 리렌더링을 방지합니다.
- **Shallow 비교**: 객체 전체를 구독해야 하는 경우 `shallow` 비교 함수를 사용하여 불필요한 업데이트를 방지합니다.

## 4. 파생 상태 (Computed Values)
- **정의**: 상태에 기반한 연산이 필요한 경우 별도의 상태로 저장하지 않고, 스토어 내부의 `get` 함수나 셀렉터 단에서 연산합니다.
- **캐싱**: 연산이 무거운 경우 `useMemo`와 결합하여 효율적으로 처리합니다.
