# Git Commit Convention Skill

이 스킬은 프로젝트의 커밋 메시지 규칙을 정의하며, 에이전트가 커밋을 수행할 때 반드시 준수해야 하는 가이드라인을 제공합니다.

## 1. 기본 원칙
- **한 커밋 = 하나의 목적**
- 현재형, 명령문으로 작성 (예: `add`, `fix`, `update`)
- 메시지는 **무엇을** 했는지 명확히 표현 (한글 사용 권장)
- 불필요한 마침표 사용 지양

## 2. 커밋 메시지 구조
```
<type>(<scope>): <subject>

<body>

<footer>
```

### 2.1 Type (필수)
| 타입 | 설명 |
| :--- | :--- |
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| refactor | 기능 변경 없는 코드 구조 개선 |
| style | 포맷팅, 세미콜론 누락 등 로직 변경 없음 |
| docs | 문서 추가 또는 수정 |
| test | 테스트 코드 추가/수정 |
| chore | 빌드 설정, 패키지 관리, 기타 작업 |
| perf | 성능 개선 |
| ci | CI 설정 변경 |
| build | 빌드 시스템 또는 외부 의존성 변경 |
| revert | 이전 커밋 되돌리기 |

### 2.2 Scope (선택)
변경 범위를 소괄호로 명시합니다. (예: `Button`, `auth`, `login-page`)

### 2.3 Subject (필수)
- 50자 이내로 명확하게 작성합니다.
- 예: `fix(auth): handle login failure gracefully`

## 3. 에이전트 자동 수행 규칙
1. 커밋 전 반드시 변경 사항을 사용자에게 브리핑하고 승인을 받습니다.
2. `AGENTS.md`에 명시된 대로 **커밋 메시지는 한글 사용**을 기본으로 하되, 타입과 스코프는 영문을 따릅니다.
3. 이슈 번호가 있는 경우 `footer`에 `Closes #이슈번호`를 포함합니다.
