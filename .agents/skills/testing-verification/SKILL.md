# Testing & Verification Skill

이 스킬은 코드 수정 후 반드시 거쳐야 하는 검증 절차와 도구를 정의합니다. 에이전트는 수정을 완료하기 전 반드시 아래 과정을 자가 수행해야 합니다.

## 1. 코드 수정 및 검증 절차
수정 완료 선언 전, 아래 명령어를 순차적으로 실행하여 에러가 없는지 확인합니다.

### 1단계: 타입 체크
```bash
pnpm type-check  # tsc --noEmit
```
- 모든 타입 오류가 해결되었는지 확인합니다.
- `any` 사용을 지양하고 구체적인 타입을 사용했는지 검토합니다.

### 2단계: 린트 및 포맷팅
```bash
pnpm lint
```
- ESLint 규칙 위반 사항이 없는지 확인합니다.
- Prettier를 통한 코드 스타일 일관성을 유지합니다.

### 3단계: 빌드 검증 (배포 전 최종 확인)
```bash
pnpm build
```
- Next.js의 프로덕션 빌드 과정에서 발생할 수 있는 잠재적 오류를 사전에 차단합니다.

## 2. 에이전트 자가 수정 지침
- 위 단계 중 하나라도 실패하면, 출력된 에러 메시지를 분석하여 스스로 수정합니다.
- 수정 후에는 다시 1단계부터 검증을 반복합니다.
- 모든 검증을 통과한 후에만 사용자에게 완료를 보고합니다.
