# Documentation Standard Skill

이 스킬은 프로젝트의 코드 가독성 향상과 문서화 품질을 유지하기 위한 가이드라인을 정의합니다. 에이전트는 코드 작성 후 나중에 사람이 읽어도 이해할 수 있도록 적절한 주석과 문서를 남겨야 합니다.

## 1. JSDoc 작성 규칙
- **함수 및 컴포넌트**: 복잡한 비즈니스 로직이나 재사용 가능한 유틸 함수, 공통 UI 컴포넌트에는 JSDoc을 작성합니다.
- **파라미터 및 반환값**: `@param`과 `@returns`를 사용하여 인자와 결과값의 의미를 명확히 설명합니다.
- **예시**:
  ```typescript
  /**
   * 사용자 생년월일을 받아 현재 나이를 계산합니다.
   * @param birthDate 'YYYY-MM-DD' 형식의 문자열
   * @returns 계산된 한국 나이
   */
  export const calculateAge = (birthDate: string): number => { ... }
  ```

## 2. 코드 내 주석 가이드라인
- **Why, Not What**: "무엇을(What)" 하는지보다는 "왜(Why)" 이 로직이 필요한지를 주석으로 남깁니다. (자명한 코드는 주석을 생략합니다.)
- **TODO/FIXME**: 임시 방편이거나 나중에 개선이 필요한 부분은 `// TODO:`, `// FIXME:` 태그를 사용하여 명시합니다.

## 3. 프로젝트 문서 업데이트
- **README.md**: 새로운 라이브러리를 추가하거나 프로젝트 실행 방식이 변경되면 즉시 `README.md`를 업데이트합니다.
- **CHANGELOG**: 중요한 기능 추가나 브레이킹 체인지가 발생하면 변경 이력을 간략히 기록합니다.

## 4. 에이전트 자동 수행 규칙
- 복잡한 알고리즘이나 독특한 설계 의도가 있는 코드를 짤 때는 반드시 JSDoc 또는 한글 주석을 남깁니다.
- 코드 수정 후 해당 파일 내에 있는 기존 주석이 여전히 유효한지 확인하고, 필요 시 업데이트합니다.
