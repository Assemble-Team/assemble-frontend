# Security & Privacy Skill

이 스킬은 프로젝트의 보안 및 개인정보 보호 가이드라인을 정의합니다. 에이전트는 작성한 코드가 보안 취약점을 만들지 않도록 극도로 주의해야 합니다.

## 1. 민감 정보 노출 방지
- **환경 변수**: `NEXT_PUBLIC_` 접두사가 붙은 환경 변수는 클라이언트에 노출되므로, API 키 등 민감 정보는 절대 사용하지 않습니다.
- **클라이언트 노출**: 클라이언트 컴포넌트로 전달하는 데이터에서 비밀번호, 주민번호 등 민감한 개인정보를 제외하고 필요한 정보만 전달합니다.

## 2. XSS (Cross-Site Scripting) 방어
- **HTML 렌더링**: `dangerouslySetInnerHTML`은 절대 사용하지 않습니다. 부득이하게 사용할 경우 반드시 `dompurify` 등을 사용하여 내용을 필터링(Sanitize)합니다.
- **사용자 입력**: 사용자로부터 받은 데이터는 그대로 화면에 출력하지 않고, 항상 React의 기본적인 이스케이핑 처리를 따릅니다.

## 3. 인증 및 세션 보안
- **토큰 저장**: `access_token`은 메모리에, `refresh_token`은 `httpOnly` 쿠키에 저장하여 XSS와 CSRF 공격을 방어합니다.
- **localStorage 사용 금지**: 세션 관리나 민감한 정보 저장을 위해 `localStorage`를 사용하지 않습니다.

## 4. 데이터 요청 보안
- **API 도메인 검증**: 외부 API 요청 시 허용된 도메인 리스트를 관리하고 이를 벗어난 요청은 사전에 차단합니다.
- **SSR 보안**: 서버 컴포넌트 내부에서 데이터를 패칭할 때, 클라이언트에서 보낸 파라미터가 적절한 유효성 검사를 거쳤는지 확인합니다.

## 5. 에이전트 자가 검토 체크리스트
- [ ] 클라이언트에 노출되는 환경 변수 중 민감한 정보가 있는가?
- [ ] 사용자 입력값이 부적절한 방식으로 렌더링되고 있는가?
- [ ] 로컬 스토리지에 민감한 정보가 저장되고 있는가?
- [ ] API 요청 시 부적절한 권한을 요구하거나 정보를 포함하고 있는가?
