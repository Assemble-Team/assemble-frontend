# Animation Strategy Skill

이 스킬은 프로젝트의 인터랙션 애니메이션 전략을 정의합니다. `framer-motion`을 사용하여 일관되고 세련된 사용자 경험을 제공하기 위한 가이드라인입니다.

## 1. 기본 애니메이션 원칙
- **감도(Duration/Easing)**: 너무 긴 애니메이션은 사용자를 지치게 합니다. 기본적으로 `0.2s` ~ `0.4s` 사이의 길이를 사용하며, 자연스러운 `easeOut` 또는 `easeInOut`을 적용합니다.
- **성능 최적화**: 레이아웃 변화(`layout` prop)는 최소화하고, 가능한 한 `transform`과 `opacity` 위주의 애니메이션을 권장합니다.
- **접근성**: `Reduced Motion` 설정을 존중하여, 시스템 설정에서 애니메이션을 줄인 사용자를 배려합니다.

## 2. 주요 애니메이션 패턴
- **컴포넌트 등장 (Entrance)**: `initial={{ opacity: 0, y: 10 }} animate={{ opacity: 1, y: 0 }}`와 같이 부드럽게 나타나는 효과를 기본으로 합니다.
- **리스트 애니메이션 (Stagger)**: 여러 아이템이 순차적으로 나타날 때는 `staggerChildren`을 사용하여 리듬감을 줍니다.
- **상태 전환 (Exit)**: 컴포넌트가 사라질 때는 `AnimatePresence`의 `exit` prop을 사용하여 갑자기 끊기지 않게 합니다.

## 3. 에이전트 자동 수행 규칙
- 새로운 컴포넌트를 만들 때, 중요한 상호작용(모달 열기, 탭 전환, 카드 호버 등)에는 세심한 애니메이션을 추가합니다.
- 애니메이션이 불필요하게 복잡해지지 않도록 주의하며, 핵심 UI 흐름을 방해하지 않는 선에서 적용합니다.
