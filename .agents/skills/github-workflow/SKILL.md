# GitHub Workflow Mastery Skill

이 스킬은 프로젝트의 GitHub 협업 워크플로우를 정의하며, 에이전트가 이슈 생성부터 PR 완료까지 품격 있게 수행하도록 돕습니다.

## 1. 이슈(Issue) 생성 가이드
- **제목**: `feat: 로그인 기능 구현`, `fix: 로그아웃 에러 수정`과 같이 명확한 타입을 포함합니다.
- **본문**: 작업할 내용(Task List), 관련 배경지식, 체크리스트를 포함합니다.

## 2. 브랜치 전략 및 체크아웃
- **네이밍**: `feat/#이슈번호-기능명`, `fix/#이슈번호-버그명` (소문자, 하이픈 사용)
- **명령어**: `git checkout -b <branch-name>`

## 3. 커밋 메시지 (다시 강조)
- **한글 사용**: 에이전트는 커밋 메시지 본문을 반드시 한글로 작성하여 변경 사항을 명확히 전달합니다.
- **이슈 연결**: 마지막 커밋 또는 PR 생성 전 `Closes #이슈번호`를 포함합니다.

## 4. PR (Pull Request) 작성 가이드
- **템플릿 활용**: `.github/pull_request_template.md`가 있다면 이를 적극 활용합니다.
- **변경 사항 요약**: 무엇을 왜 고쳤는지, 어떤 부분에 중점적으로 리뷰가 필요한지 핵심만 요약합니다.
- **이미지/비디오**: UI 변경 사항이 있다면 전/후 스크린샷이나 시각적 증거를 반드시 포함합니다. (에이전트가 직접 캡처하기 어려운 경우, 텍스트로 변경된 UI 구조를 설명합니다.)

## 5. 에이전트 자동 수행 규칙
- PR 생성 후, 사용자가 리뷰를 원활히 할 수 있도록 변경된 파일의 목록과 핵심 로직의 위치를 설명합니다.
- 커뮤니케이션 시 정중하고 전문적인 톤을 유지합니다.
