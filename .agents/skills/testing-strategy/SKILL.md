# Testing Strategy Skill

이 스킬은 프로젝트의 테스트 코드 작성 및 실행 전략을 정의합니다. 에이전트는 새로운 기능을 개발하거나 버그를 수정할 때, 관련된 테스트 코드를 작성하거나 업데이트해야 합니다.

## 1. 테스트 종류 및 도구
- **단위 테스트 (Unit Test)**: **Vitest** (또는 Jest)를 사용하여 개별 함수나 순수 로직을 테스트합니다.
- **컴포넌트 테스트 (Component Test)**: **React Testing Library (RTL)**를 사용하여 UI 컴포넌트의 렌더링과 사용자 인터랙션을 테스트합니다.
- **E2E 테스트 (End-to-End)**: **Playwright**를 사용하여 사용자 시나리오 전체를 검증합니다.

## 2. 테스트 파일 위치 및 네이밍
- **위치**: 테스트 대상 파일과 동일한 폴더 내에 배치하거나, 해당 슬라이스의 `__tests__/` 폴더를 활용합니다. (Co-location 원칙)
- **네이밍**: `[filename].test.ts` 또는 `[filename].spec.tsx` 형식을 사용합니다.

## 3. 테스트 작성 원칙
- **AAA 패턴**: Arrange(준비), Act(행동), Assert(검증) 순서로 작성하여 가독성을 높입니다.
- **Mocking**: 외부 API나 무거운 모듈은 `vi.mock` 또는 MSW를 사용하여 적절히 모킹합니다.
- **Given-When-Then**: 테스트 케이스 설명 시 "어떤 상황에서(Given), 무엇을 했을 때(When), 어떤 결과가 나와야 한다(Then)"는 논리를 따릅니다.

## 4. 에이전트 자동 수행 규칙
- 복잡한 비즈니스 로직(유틸 함수, 커스텀 훅 등)을 작성할 때는 반드시 단위 테스트를 함께 작성합니다.
- 버그 수정 시, 해당 버그를 재현하는 테스트 케이스를 먼저 작성한 후 수정을 진행합니다. (TDD 방식 권장)
