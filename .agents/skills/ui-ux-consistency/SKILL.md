# UI/UX Consistency Skill

이 스킬은 프로젝트의 디자인 일관성을 유지하기 위한 가이드라인을 정의합니다. 에이전트는 UI를 생성하거나 수정할 때 반드시 이 규칙을 준수해야 합니다.

## 1. Shared UI 우선순위 원칙
- 새로운 UI 컴포넌트를 만들기 전, 반드시 `src/shared/ui/` 폴더를 검색하여 이미 구현된 컴포넌트가 있는지 확인합니다.
- 예: 버튼이 필요하면 `Button` 컴포넌트를, 입력창이 필요하면 `Input` 컴포넌트를 우선적으로 사용합니다.

## 2. Tailwind CSS v4 스타일링 규칙
- **디자인 토큰 사용**: 임의의 색상 코드(`#ffffff`)나 수치(`px`) 사용을 지양하고, Tailwind에서 제공하는 유틸리티 클래스와 프로젝트 정의 테마를 사용합니다.
- **여백(Spacing)**: 일관된 여백을 위해 `p-4`, `m-2` 등 정해진 간격 단위를 사용하며, 레이아웃에 따라 적절한 `gap`을 설정합니다.
- **다크 모드**: 필요한 경우 다크 모드 대응을 고려하여 클래스를 작성합니다.

## 3. 접근성(A11y) 및 인터랙션
- **시맨틱 태그**: 의미 없는 `div` 남용을 지양하고, `main`, `section`, `article`, `nav` 등을 목적에 맞게 사용합니다.
- **인터랙션 피드백**: 버튼 클릭 시 `hover`, `active` 상태와 로딩 중 `disabled` 상태를 명확히 표현합니다.
- **이미지 최적화**: 모든 이미지는 `next/image`를 사용하고 적절한 `alt` 텍스트를 제공합니다.

## 4. 반응형 디자인
- 모바일 퍼스트 원칙을 준수합니다. (기본 클래스는 모바일, `md:`, `lg:` 등은 데스크톱용)
- `flex`, `grid`를 활용하여 유연한 레이아웃을 구성합니다.
